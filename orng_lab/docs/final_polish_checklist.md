# ORNG FINAL POLISH CHECKLIST

## Code quality
- No notebooks in production paths.
- One function per file.
- Explicit inputs and outputs.

## Research quality
- One hypothesis per test.
- One reason per parameter.
- One decision per result.

## Validation quality
- Baseline comparison required.
- Out of sample metrics required.
- Stress tests required.

## Risk quality
- Volatility capped.
- Drawdown explained.
- Exposure logged.

## Documentation
- Every strategy has a paper.
- Every failure has a postmortem.
- Every win has caveats.
